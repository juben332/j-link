Durian Greenhouse Automation

Monitors temperature and humidity inside a durian grow house and triggers ventilation fans via relay when thresholds are exceeded. The first DHT22 probe corroded out at week one — replaced with a waterproof DS18B20 and added a secondary unit for redundancy. Firebase Realtime Database feeds a simple dashboard. Currently tuning relay debounce logic to stop fan chatter at threshold boundaries.

Stack ESP8266 / Firebase / DHT22
Duration 3 weeks (ongoing)
Role Firmware + cloud + sensors
Status v.02 — calibration phase

No. 007 — 2024 — Embedded — solo build v.03 — stable

USB Power Meter Firmware

A USB-C inline power meter that reads voltage, current, and wattage at up to 1 kHz and displays a rolling waveform on a 0.96″ OLED. The INA226 shunt resistor caused a ground offset bug that took two PCB spins to fix — a 0.01 Ω tolerance issue. v.03 added peak-hold and a USB-PD negotiation sniffer mode. Board is 40×18 mm, fits in a standard USB-A enclosure.

Stack STM32G0 / INA226 / SSD1306
Duration 6 weeks
Role Firmware + PCB layout
Status v.03 — stable
Sec. 02 — About

I build across three disciplines — web, embedded, and hardware — and I number everything. The count is at 014. Each project gets a spec sheet, a stack list, a status, and an honest account of what broke.

The workshop direction came from the way I actually work: typed notes in a notebook, sketches on graph paper, components on a breadboard before they go on a PCB. The site is meant to reflect that — not a polished marketing surface, but a living build log.

Currently: calibrating a durian greenhouse sensor array (No. 014) and drafting a Rust-based CLI tool that parses KiCad netlists (No. 015, draft). Based in the Philippines.
